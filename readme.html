Mucho Plugin Readme

Overview

The Mucho plugin spawn users that joins chat rooms and echoes all messages to that user in that room. This could be useful for automated messaging to a JID that you want a group of people to monitor (for example some a build server or JIRA).

The plugin is a component with its own subdomain (mucho.server.com) and users are created for each chat room with the same username. Example: myroom@mucho.server.com joins the room myroom@conference.server.com. Room names are specified as server properties "plugin.mucho.ids" in a comma-separated string (example: "theuser,otheruser,thirduser").

The word "Mucho" is short for "MUC Echo" (Multi User Conference Echo).

Installation

Rename mucho-[version].jar to mucho.jar and copy it into the plugins directory of your Openfire installation. The plugin will then be automatically deployed. To upgrade to a new version, copy the new mucho.jar file over the existing file.

Configuration

The Mucho plugin is configured via Openfire system properties.

  • plugin.mucho.serviceName -- the name of the mucho service. If no value is set, the default is "mucho".
  • plugin.mucho.ids -- comma-separated list of rooms to join. Each room name will have a corresponding user in the mucho service domain. Default: "".
  • plugin.mucho.showSenderJID -- prepend each message sent by the bot with the JID of the original sender. If "true", JID is shown, otherwise not. Default: "false".
  • plugin.mucho.conferenceService -- specify conference service name. If set, it is used as conference service name, otherwise the first available is used. If no conference is avaiable "conference" is used. Default: "".
  • plugin.mucho.debug -- debug flag. If "true", verbose debug message will be written to log. Default: "false".

Using the Plugin

Join the chat room to monitor messages sent to it. You might add the user and send messages to it to make it send that message to the conference.

You can send commands to the bot from the conference by prefixing message with a "command code".

  • @sayto: <jid> <message> -- Makes the bot say "message" to "jid".

Example of usage

In this example you have setup a Cruisecontrol build server to announce new builds to a Jabber user (JID).
  1. Setup Mucho to join the room "cc" by setting the Server Property "plugin.mucho.ids" to "cc".
  2. Setup Cruisecontrol to announce to the Mucho user: "cc@mucho.server.com".
  3. Join the chat room "cc@confernce.server.com".
  4. Tell others that might want to know when cc is announcing a new build to join that chat room.
